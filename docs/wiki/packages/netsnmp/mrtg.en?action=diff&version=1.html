     17=== MRTG Config file === 
     18 
     19You can generate a initial config file with the cfgmaker tool.[[br]] 
     20This will query the snmp agent, and from the obtained info a config file is created: 
     21 perl cfgmaker public@192.168.178.1 --global "WorkDir: c:\www\mrtg" --output mrtg.cfg 
     22But be aware that I have noticed that interface index numbering changes often at reboot, or when router parameters are changed. [[br]] 
     23For the wifi interfaces 'wifi0' and 'ath0' of the 7270v3 I have even seen multiple interface indexes were created where only the last created with actual counters. A restart of the snmpd will correct this. [[br]] 
     24 
     25In the generated config file most, if not all, are commented out due to e.g. if there is no speed information found. Start with remove the comments character for one interface, and get this to work. [[br]] 
     26After you have cleared all possible config errors (e.g. watch leading spaces which seems not accepted at some locations), you can add more interfaces. [[br]] 
     27You can also change the ADSL related interfaces to have two speeds (asymmetric) so the percentage of used link speed for up and down is more realistic.

     1531 perl indexmaker --output <name>.htm mrtg.cfg

     1533 perl indexmaker --show=week --output <name>.htm mrtg.cfg 
     1534 perl indexmaker --show=month --output <name>.htm mrtg.cfg 
     1535 perl indexmaker --show=year --output <name>.htm mrtg.cfg
