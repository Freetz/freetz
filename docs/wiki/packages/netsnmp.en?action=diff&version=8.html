    104 === Installing MRTG === 
    105  
    106 I installed MRTG on a WinXP pc. For this I followed the [http://oss.oetiker.ch/mrtg/doc/mrtg-nt-guide.en.html Windows Installation Guide]. 
    107  
    108 I also installed MGTG as a service. [[br]] [[br]] 
    109  
    110  
    111 === MRTG Config file === 
    112  
    113 You can generate a initial config file with the cfgmaker tool.[[br]] 
    114 This will query the snmp agent and from the obtained info a config file is created: 
    115  perl cfgmaker public@192.168.178.1 --global "WorkDir: c:\www\mrtg" --output mrtg.cfg 
    116 But be aware that I have noticed that interface index numbering changes often at reboot, or when router parameters are changed. [[br]] 
    117 For the wifi interfaces 'wifi0' and 'ath0' I have even seen multiple interface indexes where created where only the last created with actual counters. A restart of the snmpd will correct this. [[br]] 
    118  
    119 In the generated config file most, if not all, are commented out due to e.g. if there is no speed information found. Start with commenting out one inteface and get this to work. [[br]] 
    120 After you have cleared all possible config errors (e.g. watch leading spaces which seems not accepted at some locations) you can add more interfaces. [[br]] 
    121 You can also change the ADSL related interfaces to have two speeds (asymmetic) so the precenage of used link speed for up and down is more realistic. 
    122  
    123 Here my current config: 
    124 I added config for monitoring CPU load (1, 5, and 15 minutes average), CPU Usage, and Memory usages. This to help with trouble shooting performance issues. [[br]] 
    125 A good explanantion of how to interpret the CPU load values can be found [http://blog.scoutapp.com/articles/2009/07/31/understanding-load-averages here]. [[br]]
